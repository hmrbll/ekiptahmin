# Django Admin — Module Reference

The admin lives at `/admin/` and is served in English (via `AdminLanguageMiddleware` — only this URL prefix is forced to English; the rest of the site stays Turkish).

Access requires `is_staff = True`. Full edit/delete on every model requires `is_superuser = True`.

---

## ACCOUNTS

### Users

All login accounts (regular users + admins).

| Field | What it does |
|-------|--------------|
| `username` | Display name in admin user list. Informational only — login uses email. |
| `email` | The login identifier (`USERNAME_FIELD`). Must be unique. |
| `nickname` | Name shown in app UI and the admin header (`Welcome, <nickname>`). |
| `timezone` | Default display timezone for this user (kickoffs render in their TZ). |
| `is_active` | Gates login. Set to `False` between sign-up and magic-link confirmation. |
| `is_staff` | Required for any `/admin/` access. |
| `is_superuser` | Full permissions. Bypasses model-level permission checks. |

**Common actions:**
- Promote a user to staff/superuser by ticking those flags.
- Reset a stuck signup by manually flipping `is_active = True`.
- Manually create a user (rare — most arrive via Invite).

> Note: ekiptahmin.com uses passwordless login (magic links). The Django password field exists but is not used in the regular flow. `set_unusable_password()` is called on sign-up.

### Invites

One-time signup links generated by an admin and shared out-of-band (WhatsApp, e-mail, etc).

| Field | What it does |
|-------|--------------|
| `code` | Auto-generated 22-char URL-safe token. Read-only. |
| `email` | Optional. Pre-fills the sign-up form for this address; signup form rejects mismatched emails. |
| `note` | Internal label, e.g., "for Ahmet" — appears in the invite list. |
| `expires_at` | Default: 30 days from creation. After this the invite link returns 410 Gone. |
| `used_at`, `used_by` | Auto-filled when the recipient confirms their magic link. |

The invite list view shows a colored **status badge**:

- 🟢 Active — usable
- ⚪ Used — already redeemed
- 🔴 Expired — past `expires_at`

**Invite link** column shows the full clickable URL (`https://ekiptahmin.com/invite/<code>/`).

**Workflow:**
1. Click *Add Invite* → optionally fill `email`/`note` → Save.
2. Copy the generated **Invite link** from the detail view.
3. Send it to your friend.
4. They click → signup form → magic link → confirm → invite is consumed (`used_at` set).

> The invite is **only consumed when the magic link is confirmed**, not on form submit. A typo in the email doesn't burn the invite.

## TOURNAMENT

### Tournaments

Top-level container. Each tournament (WC 2026, EURO 2028, ...) is a separate row, so the same project can host multiple tournaments without data conflicts.

| Field | What it does |
|-------|--------------|
| `name` | e.g., "FIFA Dünya Kupası 2026". User-facing in the UI. |
| `slug` | URL-safe identifier, used by the seed command (`wc2026`). |
| `start_date`, `end_date` | Tournament span. Cosmetic; doesn't gate anything. |
| `is_active` | Drives the default UI selection. Only one tournament should be active at a time. |
| `timezone` | Fallback display TZ for anonymous visitors (logged-in users use their own). |

The Tournament edit page shows **Stages** and **Prediction rounds** as inlines — set up everything in one place when bootstrapping a new tournament.

### Stages

Tournament phases. Each stage holds **two scoring configurations** — ganyan pools (active) and legacy bracket points (staff-only `/legacy/*` views).

| Field | What it does |
|-------|--------------|
| `kind` | Enum: GROUP, R32, R16, QF, SF, THIRD, FINAL |
| `order` | 0 = Group, 6 = Final. Drives sorting. |
| **Ganyan pools (active engine)** | |
| `pool_exact` | Pool size for the "exact score" criterion. Split equally among users who get it right. Default 100. |
| `pool_diff` | Pool size for "goal difference". Default 100. |
| `pool_result` | Pool size for "outcome (1X2)". Default 100. |
| `pool_penalty_winner` | Pool for "named the team that advanced via penalties" (KO only). Default 50. |
| `pool_penalty_score` | Pool for "exact penalty shootout score" — draw predictions only (KO). Default 50. |
| `pool_penalty_diff` | Pool for "penalty shootout goal difference" — draw predictions only (KO). Default 50. |
| **Legacy bracket scoring (staff-only views)** | |
| `points_exact` | LEGACY. Points for correct exact score. |
| `points_diff` | LEGACY. Points for correct outcome + goal difference. |
| `points_result` | LEGACY. Points for correct outcome only. |
| `penalty_loser_pct` | LEGACY. Percentage bonus when user named the penalty advancer despite predicting a non-draw. |

All scoring fields are inline-editable in the Stage list view. The Stage detail page collapses the legacy section so the active pools are the foreground.

> All scoring values are admin-tunable on purpose. There's no hardcoded scoring anywhere in Python code. Pool sizes you edit here **persist across deploys** — `seed_wc2026` writes them only when a Stage is first created, so it never overwrites your edits. After editing pool sizes, run `python manage.py recompute_ganyan` to refresh the cache (signals don't fire on Stage saves).

The two engines run side by side on every `ActualResult` write. Pool sizes drive the public site; the legacy points drive `/legacy/scoring-diff/` for calibration.

### Teams

Tournament participants. Tournament-scoped (the same country in two tournaments is two separate rows).

| Field | What it does |
|-------|--------------|
| `code` | FIFA 3-letter code (TUR, BRA, ESP, ...). Used by seed CSVs and forms. Drives the SVG flag lookup via the `FIFA_TO_ISO` mapping on the model. |
| `name_tr` | Country name in Turkish — appears in dropdowns, leaderboard, predictions. |
| `flag_emoji` | Fallback flag emoji (Linux servers usually lack emoji fonts, so SVG is the primary display). |
| `group_letter` | A–L for the 12 groups. Empty for any non-group-stage team. |

Filter by `tournament` and `group_letter` in the list view; search by `name_tr` or `code`.

**Flags:** The team list shows SVG flags from [lipis/flag-icons](https://github.com/lipis/flag-icons) (MIT, in `static/flags/`). To add a new team's flag, add the FIFA→ISO entry to `Team.FIFA_TO_ISO` and drop the SVG into `static/flags/<iso>.svg`.

### Prediction rounds

Defines the multi-round prediction mechanic. There are 7 rounds for WC 2026 (one before kickoff, then one after each knockout phase), each with its own deadline and weight.

| Field | What it does |
|-------|--------------|
| `name` | e.g., "Pre-turnuva", "Grup sonrası". User-facing. |
| `order` | 0–6. Drives the round flow. |
| `deadline` | UTC. After this, predictions for this round can no longer be submitted/edited. |
| `weight` | Score multiplier (e.g., 1.00 → 0.85 → 0.75 → ...). Earlier rounds carry higher weight. |
| `editable_stages` | M2M to Stage — which stages a user can submit/edit predictions for in this round. |

The `is_open` column shows live whether the round is currently accepting predictions.

> **Rounds are admin-owned after creation.** `seed_wc2026` only creates rounds that don't exist yet — a deploy never overwrites `deadline`, `editable_stages`, or any other round field. This is what makes mid-tournament edits safe: removing GROUP from a round's `editable_stages` closes group predictions for good, and an extended deadline stays extended.

> Round weight encodes "early predictions are riskier and worth more". A correct matchup with the same score predicted in Round 1 (weight 1.00) earns more than the same matchup predicted in Round 4 (weight 0.65). The scoring engine evaluates every round in which the user predicted the matchup correctly and picks the one that yields the highest weighted total; ties go to the earlier round. A user who got the matchup right early with a wrong score and then fixed the score later is no longer locked out of points.

### Bracket slots

The structural backbone — each row is a single match position. 104 total for WC 2026 (72 group + 32 knockout).

**Position naming convention** (stable IDs, used everywhere):

| Stage | Pattern | Example |
|-------|---------|---------|
| Group | `Group<L>-M<n>` | `GroupD-M3` |
| R32 | `R32-<n>` | `R32-7` |
| R16 | `R16-<n>` | `R16-1` |
| QF | `QF-<n>` | `QF-3` |
| SF | `SF-<n>` | `SF-2` |
| Third | `Third` | `Third` |
| Final | `Final` | `Final` |

| Field | What it does |
|-------|--------------|
| `position` | The stable string ID. Don't change after seed unless you really know what you're doing. |
| `stage` | FK to Stage, drives scoring rules. |
| `scheduled_kickoff` | UTC. Predictions auto-lock once this passes. |
| `venue` | Stadium + city, free text. |
| `home_team_actual`, `away_team_actual` | Filled by seed for group matches. For knockout slots, populated as the bracket resolves (later, in Phase 4 logic). |
| `home_source`, `away_source` | Free-text descriptions for unresolved knockout slots, e.g., "A Grubu 1.si" or "R32-1 Kazananı". Shown in the bracket UI. |

**Inline:** Actual result (one-to-one). Add the result directly from the slot detail page.

**Locked column** in the list view shows whether predictions are frozen for that slot.

### Actual results

The match results, entered by an admin once the match is over. Triggers scoring once present.

| Field | What it does |
|-------|--------------|
| `slot` | Link to the BracketSlot this is the result for (one-to-one). |
| `home_score`, `away_score` | 90-minute regulation score. This is the canonical scoring basis (matches the 2022 group rules). |
| `went_to_extra_time` | Informational flag. |
| `went_to_penalties` | When True, the penalty fields below populate. Ganyan engine activates the three penalty pools (`pool_penalty_winner` / `pool_penalty_score` / `pool_penalty_diff`); legacy engine applies the "penalty loser bonus" rule. |
| `penalty_winner`, `home_penalties`, `away_penalties` | Penalty shootout details (only when `went_to_penalties` is True). |
| `entered_at` | Auto-stamped on save. |
| `entered_by` | Auto-filled to the admin user who saved the row. |

**Workflow:**
- After a match finishes, find its slot via Bracket slots → click the slot → scroll to the inline ActualResult section → fill in scores → save.
- Or use the standalone **Actual results** list view to bulk-enter via search-by-position.

## SCORING

### Ganyan scores (`GanyanScore`)

Materialized per-(user, slot) cache from the active parimutuel engine. Read-only listing for staff investigation — rebuilds happen via signals on `ActualResult` and `SlotPrediction` writes.

### Match pools (`MatchPool`)

One row per (slot, criterion). Stores pool size snapshot, predictor/winner counts, base payout, and the breakdown JSON used by the ganyan tablosu UI. Read-only.

### Slot scores (`SlotScore`)

LEGACY bracket engine cache. Still updated alongside `GanyanScore` so the staff-only `/legacy/leaderboard/` and `/legacy/scoring-diff/` views work.
